🏥 HOSDT: Healthcare Operations Simulation & Diagnostic ToolkitHOSDT is a high-fidelity Python-based simulation engine designed to model the operational flow of a single-doctor clinic. It uses sequential queue logic to simulate realistic patient behavior, administrative bottlenecks, and financial performance.🚀 OverviewThis project provides a "Command Center" view for clinic administrators or consultants to understand how different variables (patient load, doctor speed, pharmacy conversion) impact the overall efficiency and revenue of a healthcare facility.Key FeaturesSequential Queue Logic: Unlike simple random generators, HOSDT calculates wait times based on when the doctor actually becomes free from the previous patient.Realistic Chaos: Simulates peak-hour arrivals, registration delays for new vs. returning patients, and age-based service durations.Financial Modeling: Tracks consultation fees and pharmacy spend with conditional probability (e.g., elderly patients are more likely to visit the pharmacy).Interactive Dashboard: Built with Streamlit and Plotly for real-time "What-If" scenario testing.🛠️ Tech StackLanguage: Python 3.10+Dashboard: StreamlitData Analysis: Pandas, NumPyVisualizations: Plotly Express & Graph ObjectsEnvironment: Optimized for GitHub Codespaces (Potato PC friendly)📊 Core Metrics TrackedWait Duration: Time spent from the end of registration until the doctor is free.Reg Duration: Administrative time taken for intake (differentiated by patient type).Pharmacy Conversion: Percentage of patients purchasing medication post-consultation.Hourly Traffic Density: Identification of "Red-line" hours where the doctor is over-capacity.🏃 Setup InstructionsTo run this on your local machine or in GitHub Codespaces:Clone the repository:git clone [https://github.com/YOUR_USERNAME/HOSDT-Clinic-Sim.git](https://github.com/YOUR_USERNAME/HOSDT-Clinic-Sim.git)
